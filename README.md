# Спасение Ковчега

Реал-тайм мультиплеерная образовательная игра по информатике (8–11 классы).
Команды захватывают секторы космического ковчега, решая задачи. Весь UI/контент — на русском.

**Стек:** Node + Express + Socket.io (сервер, порт 3001) · React + Vite + Tailwind (клиент, порт 5173).

---

## Требования
- **Node.js 18+** и npm (проверка: `node -v`, `npm -v`).
- Любая ОС (Windows / macOS / Linux).

## Установка
В двух отдельных терминалах из корня проекта:

```bash
# 1) Сервер
cd server
npm install

# 2) Клиент
cd client
npm install
```

## Запуск (режим разработки)
Два терминала:

```bash
# Терминал A — сервер (http://localhost:3001)
cd server
npm run dev

# Терминал B — клиент (http://localhost:5173)
cd client
npm run dev
```

Затем откройте в браузере **http://localhost:5173**.

> macOS/Linux: можно одной командой `./start.sh` из корня (поднимет оба процесса).

## Вход
- **Учитель:** на стартовом экране «Учитель» → логин/пароль. Резервный вход: логин `admin`,
  пароль `aegis-teacher` (меняется переменной окружения `ADMIN_PASSWORD`). Можно
  зарегистрировать своего учителя на вкладке «Регистрация».
- **Игрок:** «Игрок» → ввести позывной и 6-значный код комнаты, который показывает учитель.

## Как играть (коротко)
1. Учитель входит и создаёт комнату → получает код.
2. Игроки заходят по коду, создают/выбирают команды (первый в команде — капитан).
3. Учитель в лобби выбирает **длительность**, **режим** («Отборочный тур» — у каждой
   команды свой корабль / «Финал» — общая карта) и **пул задач** (можно импортировать пак),
   затем запускает игру.
4. Команды решают задачи и захватывают секторы. Победа — по очкам к концу таймера
   или при зачистке всех секторов.

## Производственная сборка (опционально)
Сервер умеет раздавать собранный клиент сам:

```bash
cd client && npm run build      # создаёт client/dist
cd ../server && npm start       # отдаёт client/dist на http://localhost:3001
```

> `client/dist` и `node_modules` не входят в архив/репозиторий — их создаёт `npm install`/`npm run build`.

## Заметки
- Состояние комнат/матчей хранится в памяти сервера — при перезапуске сбрасывается.
  Профили игроков и кастомные задачи персистятся в JSON-файлах в `server/`.
- Известные баги и недоделки — в `.troubles.md`.
